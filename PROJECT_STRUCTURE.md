# Fathom - Project Structure

## Directory Layout

```
Fathom/
├── .git/                    # Git repository
├── .vscode/                 # VS Code settings
├── Assets/                  # Unity assets (all game content)
│   ├── Scenes/             # Unity scene files
│   ├── Scripts/            # C# scripts
│   │   ├── Core/          # Core game systems
│   │   ├── Player/        # Player-related scripts
│   │   ├── Fish/          # Fish AI and behavior
│   │   ├── UI/            # UI controllers
│   │   └── Managers/      # Game managers (singletons)
│   ├── Prefabs/           # Reusable game objects
│   │   ├── Player/        # Player prefabs
│   │   ├── Fish/          # Fish prefabs
│   │   ├── UI/            # UI prefabs
│   │   └── Environment/   # Environment prefabs
│   ├── Sprites/           # 2D graphics
│   │   ├── Player/        # Player sprites
│   │   ├── Fish/          # Fish sprites
│   │   ├── Environment/   # Background, terrain
│   │   └── UI/            # UI graphics
│   ├── Audio/             # Sound and music
│   │   ├── Music/         # Background music
│   │   └── SFX/           # Sound effects
│   └── Resources/         # Runtime-loaded assets
├── Library/               # Unity cache (ignored by git)
├── Logs/                  # Unity logs (ignored by git)
├── Packages/              # Unity package manager
├── ProjectSettings/       # Unity project configuration (committed)
├── Temp/                  # Temporary files (ignored by git)
├── UserSettings/          # User-specific settings (ignored by git)
├── .gitignore             # Git ignore rules
├── .gitattributes         # Git LFS configuration
├── Fathom.sln             # C# solution file (generated by Unity)
├── README.md              # Project overview
└── DEVELOPMENT_PLAN.md    # Development roadmap
```

## Unity Version
- **Engine:** Unity 2022.3.62f3 LTS
- **Template:** Universal 2D (URP)

## IDE Setup
- **Editor:** Visual Studio Code
- **Extensions Required:**
  - Unity for Visual Studio Code
  - C# Dev Kit

## Getting Started

### Opening the Project
1. Open Unity Hub
2. Click "Add" → Select this folder
3. Click on "Fathom" to open in Unity Editor

### Opening in VS Code
- **Option 1:** Open the entire `Fathom` folder in VS Code
- **Option 2:** Double-click any `.cs` script in Unity (will open in VS Code if configured)

### Configuring Unity to Use VS Code
1. In Unity Editor: **Edit → Preferences** (Mac: **Unity → Settings**)
2. Go to **External Tools**
3. Set **External Script Editor** to **Visual Studio Code**
4. Click **Regenerate project files**

## Naming Conventions

### Scripts
- **Classes:** PascalCase (e.g., `PlayerController`, `FishAI`)
- **Methods:** PascalCase (e.g., `UpdateMovement()`)
- **Variables:** camelCase (e.g., `swimSpeed`, `currentOxygen`)
- **Private fields:** camelCase (e.g., `targetPosition`)
- **Constants:** UPPER_SNAKE_CASE (e.g., `MAX_DEPTH`)

### Files and Folders
- **Folders:** PascalCase (e.g., `Scripts`, `Prefabs`)
- **Scene files:** PascalCase (e.g., `MainMenu.unity`, `Game.unity`)
- **Prefabs:** PascalCase with descriptor (e.g., `Fish_Passive.prefab`, `Player.prefab`)

## Version Control

### What's Committed
- `/Assets` (all game content)
- `/ProjectSettings` (Unity configuration)
- `/Packages` (package manifest)
- Root files (README, .gitignore, etc.)

### What's Ignored
- `/Library` (Unity cache - regenerates)
- `/Temp` (temporary build files)
- `/Logs` (Unity logs)
- `/UserSettings` (user-specific settings)
- `/obj` and `/bin` (C# build artifacts)
- `*.csproj`, `*.sln` (regenerated by Unity)

### Git LFS
Large binary files tracked by Git LFS:
- Images: `.png`, `.jpg`, `.psd`
- Audio: `.wav`, `.mp3`, `.ogg`
- 3D Models: `.fbx`, `.obj`
- Unity Assets: `.unity`, `.prefab`, `.asset`
